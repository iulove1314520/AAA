# Linux系统管理脚本

一个功能全面的Linux系统管理脚本，提供系统信息查看、系统配置管理、网络管理和Docker管理等功能。

## 主要功能

### 1. 系统信息
- **基本系统信息**
  - 操作系统版本
  - 内核版本
  - 系统时间和时区
  - 运行时间
  - 用户信息
- **CPU信息**
  - CPU型号和架构
  - 核心数和线程数
  - CPU使用率
- **内存信息**
  - 物理内存使用情况
  - Swap使用情况
  - 内存使用TOP进程
- **磁盘信息**
  - 磁盘使用情况
  - 分区信息
  - I/O统计
- **网络信息**
  - 网络接口状态
  - 路由表
  - DNS配置
  - 网络连接统计

### 2. 系统配置管理
- **用户管理**
  - 添加/删除用户
  - 修改用户密码
  - 管理用户组
  - 查看用户详情
- **时区管理**
  - 查看当前时区
  - 修改系统时区
  - 同步系统时间
  - 修改时间格式
- **主机管理**
  - 查看/修改主机名
  - 管理hosts文件
  - 备份/恢复hosts配置
- **Swap管理**
  - 查看Swap状态
  - 创建/删除Swap文件
  - 调整Swap大小
  - 修改Swap优先级
- **网络加速**
  - BBR加速安装
  - TCP拥塞控制
  - 网络参数优化
- **系统服务**
  - 服务启动/停止
  - 开机启动设置
  - 服务状态查看
  - 服务日志查看
- **系统更新**
  - 更新软件包
  - 清理系统
  - 自动更新设置
- **软件包管理**
  - 安装/卸载软件
  - 修复损坏包
  - 清理缓存

### 3. 网络管理
- **防火墙管理**
  - 防火墙状态控制
  - 端口管理
  - 规则管理
  - 开机启动设置
- **IP协议管理**
  - IPv4/IPv6设置
  - IP转发配置
  - IP地址管理
- **DNS配置**
  - 查看/修改DNS
  - DNS服务器管理
  - 配置备份/恢复
- **路由管理**
  - 查看路由表
  - 静态路由配置
  - 默认网关设置
- **网络测试工具**
  - Ping测试
  - 路由追踪
  - DNS查询
  - 端口扫描
  - 带宽测试
  - 网络抓包

### 4. Docker管理
- **Docker安装/更新**
  - 检查Docker状态
  - 安装Docker
  - 更新Docker
  - Docker Compose安装
- **容器管理**
  - 查看容器列表
  - 启动/停止容器
  - 容器日志查看
  - 进入容器终端
- **镜像管理**
  - 查看本地镜像
  - 拉取/删除镜像
  - 镜像导入导出
  - 镜像构建
- **网络管理**
  - 创建/删除网络
  - 容器网络连接
  - 网络驱动配置
  - 网络连接测试
- **数据卷管理**
  - 创建/删除数据卷
  - 数据卷备份
  - 数据卷恢复
  - 清理未使用卷
- **Docker配置**
  - 镜像加速器设置
  - 存储目录修改
  - 日志配置
  - DNS配置
- **Docker Compose**
  - 项目启动/停止
  - 查看项目日志
  - 更新项目镜像
  - 项目配置管理

## 使用说明

### 安装
```bash
# 下载脚本
git clone https://github.com/yourusername/yourrepo.git
cd yourrepo

# 添加执行权限
chmod +x manage.sh
```

### 运行
```bash
# 需要root权限运行
sudo ./manage.sh
```

### 菜单导航
- 使用数字键选择功能
- 按0返回上级菜单
- 按Ctrl+C退出脚本

## 系统要求
- 支持的操作系统：CentOS 7/8, Debian 10+, Ubuntu 18.04+
- 需要root权限
- 需要基本的网络连接
- Docker管理功能需要安装Docker

## 注意事项
1. 首次运行会自动检查并安装必要的工具
2. 修改系统配置前建议先备份
3. Docker相关操作需要Docker服务正常运行
4. 某些功能可能需要额外的软件包支持
5. 建议在测试环境中先测试后再在生产环境使用

## 常见问题
Q: 为什么需要root权限？  
A: 脚本需要修改系统配置和管理Docker，必须使用root权限。

Q: 如何更新Docker？  
A: 在Docker管理菜单中选择"安装/更新Docker"选项。

Q: 数据卷备份在哪里？  
A: 数据卷备份文件保存在用户指定的路径中。
